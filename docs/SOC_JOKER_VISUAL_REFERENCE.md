## Interaction Flow

### Playing a Card

```
1. User clicks card in hand
   └─> Card is highlighted (blue glow)
   
2. User clicks threat
   └─> Card effect is applied
       ├─> Damage dealt to threat
       ├─> Threat health bar updates
       ├─> Card removed from hand
       └─> If threat health <= 0:
           └─> Threat removed from play
               └─> If all threats defeated:
                   └─> Wave complete!
```

### End Turn Sequence

```
1. User clicks "End Turn" button
   
2. For each threat:
   ├─> Threat attacks player
   └─> Player health reduced
   
3. Check if player health <= 0
   ├─> YES: Run failed (defeat screen)
   └─> NO: Continue turn
   
4. Discard current hand
   
5. Draw 5 new cards
   └─> If draw pile empty:
       └─> Shuffle discard pile into draw pile
```

### Shop Sequence

```
1. Wave completes successfully
   
2. Currency awarded based on wave
   
3. 3 random cards displayed
   
4. User clicks "Buy" on desired card
   ├─> Check if enough currency
   │   ├─> YES: 
   │   │   ├─> Deduct currency
   │   │   ├─> Add card to deck
   │   │   └─> Refresh shop display
   │   └─> NO: Show "Too Expensive"
   
5. User clicks "Continue to Next Wave"
   └─> Next wave begins
```

## Design Notes

- **Minimalist**: Focus on card gameplay, not clutter
- **Clear feedback**: Every action has visual response
- **Mobile-friendly**: Large touch targets (buttons/cards)
- **Accessibility**: High contrast, clear text
- **Responsive**: Scales to different resolutions
- **Performance**: Efficient rendering, no lag
